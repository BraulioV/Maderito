Comunicación entre Raspberry Pi y Arduino

  • La comunicación se realiza mediante el puerto serie.
  • En Python usamos la librería pyserial.
  • En Arduino, la clase Serial.
  • Arduino envía una m (moving) o una q (quiet).
  • Python envía una l (left), una r (right) o una q (quiet).

Problemas

  • Los cables hembra que teníamos no enganchaban bien con los CNY70, así que tuvimos que soldarlos.
  • Los motores que teníamos no tenían mucha fuerza, tuvimos que pensar en una estrategia diferente a la de empujar.
  • La Raspberry no tenía potencia suficiente como para hacer estereoscopía, tuvimos que usar sólo una cámara.

Problemas

  • Cuando comunicábamos ambas placas, teníamos que limpiar el buffer de vídeo para poder reconocer correctamente cuando se movía el contrario.
  • OpenCV reconocía objetos del fondo, tuvimos que poner la cámara en alto para que el único fondo fuese el tatami.
